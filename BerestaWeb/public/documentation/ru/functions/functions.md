# Синтаксис функций

Функции в Beresta Language позволяют объединять повторно используемые части логики.  
Они могут принимать параметры, возвращать значения и объявляться как `public` или `private`.

---

## Синтаксис
```beresta
function name(parameters)
{
    // код
    return value;
}
```
Можно также явно указать область видимости:
```beresta
public function name(parameters) {}
private function name(parameters) {}
```

---

## Описание

- Публичные функции (`public`) могут вызываться из любого файла или модуля.  
- Приватные функции (`private`) доступны только внутри файла, где они были объявлены.  
- Оператор `return` возвращает значение вызывающему коду. Если значение не указано, функция всё равно выполняется и возвращает `undefined`.

Функции являются объектами первого класса — их можно присваивать переменным, возвращать из других функций и вызывать внутри выражений.

---

## Примеры

Базовая функция:
```beresta
public function add(a, b)
{
    return a + b;
}

let answer = add(2, 3);
console_print(answer); // 5
```

Функция без возвращаемого значения:
```beresta
private function greet(name)
{
    console_print("Hello " + name); // Привет, Denis
}

let x = greet("Denis");
// вывод: Hello Denis
// переменная x содержит undefined
```

Использование функций внутри выражений:
```beresta
function double(x)
{
    return x * 2;
}

console_print(double(10) + 5); // 25
```

---

## Модификаторы области видимости

| Модификатор | Описание |
| --- | --- |
| `public` | Функция может быть вызвана из любого файла или модуля. |
| `private` | Функция доступна только в пределах файла, где она объявлена. |

Если модификатор не указан, функция по умолчанию считается `public`.

---

## Вызов внешних функций

Публичные функции из других файлов можно вызывать напрямую по имени.  
Пример:
```beresta
let result = calculate(5, 10, 2); // функция должна быть в другом файле
console_print(result);
```

---

## Примечания

- Параметры функций имеют динамическую типизацию.  
- Количество переданных аргументов может отличаться от количества параметров.  
- Если передано меньше аргументов, недостающие параметры получают значение `undefined`.  
- Функции могут быть вложенными или рекурсивными (см. следующую секцию о рекурсии).
